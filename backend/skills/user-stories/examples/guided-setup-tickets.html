Description

What

Build the stepwise tracker-connection flow — not-started → authorizing → verifying → connected — with progress that persists across sessions and OAuth interruptions.

Why now

Connection is where new accounts die: it's the top VoC theme (11 of 17 calls, 19 of 42 tickets), and 11 of 15 accounts stalled more than a week before their first agent run. This is the walking skeleton — everything else in the PRD builds on a connection that works.

User story

As a setting-up PM, I want a guided connection flow that shows me where I am and never loses my progress, so that setup is something I finish, not something I abandon.

The flow must cover
  • The four explicit states, always visible, in order
  • Progress persisted across session breaks
  • OAuth cancel/expiry returning to a resumable state — never a restart
  • Launch trackers per DT-2 (Jira + Linear confirmed; Asana pending)
Out of scope

Failure-state UX (GS-3) · the starter run (GS-4) · admin visibility (GS-5).

Grounding: PRD § 5 R1 · activation baseline [NEED — GS-1 creates it]

Acceptance criteria — 4

Given a user initiating tracker connection, When they proceed, Then the flow presents not-started → authorizing → verifying → connected in order, every state visible.
[failure] Given a session break mid-flow, When the user returns, Then progress is preserved — a skipped or lost state fails this ticket.
[edge] Given an OAuth authorization cancelled or expired mid-flow, When the user resumes, Then they return to a resumable state with prior steps preserved.
[failure] Given the same OAuth interruption, When resume would restart the user from zero, Then the test fails.
